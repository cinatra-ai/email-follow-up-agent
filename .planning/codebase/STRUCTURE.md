## Directory Purposes

**`cinatra/`:**
- Purpose: Cinatra platform sidecar directory — the runtime reads artifacts from here
- Contains: `oas.json` (agent flow graph in OAS/JSON format)
- Key files: `cinatra/oas.json`

**`skills/email-follow-up/`:**
- Purpose: Skill package loaded by the Cinatra runtime when `agent_id: "email-follow-up"` is resolved
- Contains: `SKILL.md` (Markdown with YAML front-matter; defines LLM system prompt behaviour)
- Key files: `skills/email-follow-up/SKILL.md`

**`.github/workflows/`:**
- Purpose: GitHub Actions CI/CD definitions
- Contains: `ci.yml` (build + kind-specific gate), `release.yml` (package publish)
- Key files: `.github/workflows/ci.yml`

**`.planning/codebase/`:**
- Purpose: GSD codebase mapper output; consumed by `/gsd-plan-phase` and `/gsd-execute-phase`
- Generated: Yes (by GSD mapper)
- Committed: Yes

## Key File Locations

**Entry Points:**
- `cinatra/oas.json`: Agent flow definition; `start_node` references `start` component
- `extension-kind-gate.mjs`: CI gate entry point; `main()` at line 365

**Configuration:**
- `package.json`: Package name, version, license, `cinatra` metadata block (kind, agentDependencies)
- `tsconfig.json`: TypeScript compiler options (targets `src/`, outputs to `dist/`)
- `.npmrc`: npm/pnpm registry configuration

**Core Logic:**
- `cinatra/oas.json`: All runtime agent behaviour (nodes, data/control edges, LLM prompt templates)
- `skills/email-follow-up/SKILL.md`: LLM instruction set (steps 1–4, quality standards, MCP tool list)
- `extension-kind-gate.mjs`: Pre-publish validation logic (`validateAgent`, `validateWorkflow`, `validateBpmnSanity`)

**Testing:**
- No test files present; the repo is a content-only source mirror — tests run in the cinatra monorepo

## Naming Conventions

**Files:**
- Cinatra sidecar artifacts: lowercase with extension matching platform contract (`oas.json`, `workflow.bpmn`)
- Gate scripts: kebab-case with `.mjs` extension (`extension-kind-gate.mjs`)
- Skill files: `SKILL.md` (uppercase, Markdown)
- CI workflows: lowercase kebab-case (`ci.yml`, `release.yml`)

**Directories:**
- Platform sidecar: `cinatra/` (fixed name, required by runtime)
- Skills: `skills/<skill-id>/` where `<skill-id>` matches `agent_id` used in the OAS node
- Planning: `.planning/codebase/` (GSD convention)

## Where to Add New Code

**New agent flow node:**
- Add node definition to `cinatra/oas.json` under `$referenced_components`
- Add `$component_ref` entry to `nodes` array
- Add `ControlFlowEdge` entries to `control_flow_connections`
- Add `DataFlowEdge` entries to `data_flow_connections` for any data passing

**New or updated skill instructions:**
- Edit `skills/email-follow-up/SKILL.md`
- Keep YAML front-matter (`name`, `description`) intact
- Follow existing section headings: Inputs, Steps, quality standards, MCP tool list

**New TypeScript source (if src/ is introduced):**
- Place under `src/` (referenced by `tsconfig.json` as `rootDir`)
- Output goes to `dist/` (gitignored by convention)

**New CI validation rule (banned primitive):**
- Add token to `BANNED_PRIMITIVES` or `BANNED_TYPEHINTS` arrays in `extension-kind-gate.mjs` (lines 65–74)
- Keep the gate zero-dependency (Node.js builtins only)

## Special Directories

- Committed: Yes

**`dist/`** (not present):

- Generated: Yes
